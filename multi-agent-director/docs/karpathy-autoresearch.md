## 3. 十大发现 (Karpathy + 社区)

### 发现 1: 更多 Step 始终优于更大 Batch

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  实验数据:                                                                  │
│  batch_size: 2^19 → 2^18 (减半)                                           │
│  训练步数: 加倍                                                             │
│  结果: BPB (Bits Per Byte) 改善了 0.007                                    │
│                                                                             │
│  结论: 小 batch + 多 step > 大 batch + 少 step                            │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 发现 2: 简单的注意力模式就是最好的

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  最终收敛到的架构: SSSL                                                     │
│  • 3 个短上下文层                                                          │
│  • 1 个长上下文层                                                          │
│  • 重复                                                                   │
│                                                                             │
│  教训:                                                                     │
│  ✗ 过多长层 → 浪费计算在全局注意力上                                       │
│  ✗ 过少长层 → 跨 token 信息缺失                                            │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 发现 3: 调整初始化比调整优化器更重要

**三项改动带来 ~0.004 BPB 改善:**

1. Value embedding 使用正态初始化
2. QKV 缩放倍率
3. 残差连接加上可学习权重

> 在大模型预训练中，0.001 都算有效改进

### 发现 4: 能学习的就别写死

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  案例:                                                                     │
│  • skip-2 残差权重 → 可学习                                                │
│  • 残差混合的 lambda 系数 → 可学习                                          │
│  • value embedding 的门控参数 → 可学习                                      │
│                                                                             │
│  效果: 5 分钟短训练也能收敛并产生收益                                       │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 发现 5: 最优架构出人意料地小

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  群体探索结果:                                                              │
│  最优配置: 12 层, 维度 512, aspect ratio 40                              │
│                                                                             │
│  16 层实验:                                                                │
│  • 参数增加 84%                                                           │
│  • 步数减少 23%                                                            │
│  • BPB 反而更差                                                            │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 发现 6: 大量"改进"其实是噪声

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  关键实验:                                                                 │
│  智能体跑了 100 组随机种子实验                                             │
│                                                                             │
│  发现: 种子方差 ≈ 0.002 BPB                                               │
│                                                                             │
│  结论: 很多声称的"改进"只是运气好                                          │
│                                                                             │
│  后果: 智能体自发调整行为:                                                 │
│  ✓ 要求重复实验                                                           │
│  ✓ 多种子验证                                                             │
│  ✓ 独立确认                                                               │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 发现 7: 一些公认好技术直接翻车

| 技术 | 结果 | BPB |
|------|------|-----|
| Weight tying | 灾难性退化 | 3.216 |
| Label smoothing | 退化 | 1.32 |
| PaLM 风格 z-loss | 一致性退化 | - |

> 这些负面结果成为集群最有用的知识，后来的智能体自动避开

### 发现 8: 最大的机会可能还没智能体碰

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  实验分布:                                                                 │
│  • 架构改动: 1045 次实验                                                   │
│  • 数据管道: 0 次实验 (1000+ 假设未被测试)                                 │
│                                                                             │
│  假设包括:                                                                 │
│  • 课程学习                                                                │
│  • 数据排序                                                                │
│  • 领域特定批处理                                                          │
│                                                                             │
│  结论: 最大突破可能在数据调度，而非架构                                    │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 发现 9: 集体记忆加速了发现过程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  机制:                                                                     │
│ 后来的智能体可以直接从已知最优配置出发                                      │
│ 不用从头重新发现前人的工作                                                 │
│                                                                             │
│ 效果:                                                                      │
│ 关键突破来自综合已有结果而非盲目探索的智能体                               │
│ 共享记忆显著加速研究进程                                                   │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 发现 10: 为了优化，智能体"不择手段"

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  有趣时刻:                                                                 │
│                                                                             │
│  1. AI 上网搜答案                                                           │
│  → 从别人开源仓库抄最优解                                                  │
│                                                                             │
│  2. 容差感知优化                                                           │
│  → 读懂评估器源代码，在规则边界内继续推进                                  │
│                                                                             │
│  3. 删除优化器                                                             │
│  → 从零写一个新的 AdamW                                                    │
│                                                                             │
│  关键区别:                                                                  │
│  传统超参搜索: 在预设范围内调数字                                          │
│  Autoresearch: 可以改变整个搜索空间                                        │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 4. 对 Multi-Agent 导演系统的启发

### 整合 Karpathy 思想

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    Multi-Agent 导演系统 + Autoresearch                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. 自发分工                                                               │
│  ┌─────────────────────────────────────────────────────────────────┐       │
│  │ 🎬 总导演 (Orchestrator) - 统筹协调                             │       │
│  │                                                                  │       │
│  │ 新增角色:                                                        │       │
│  │ • 📊 统计员 - 测量创意方差、置信度                              │       │
│  │ • 🔄 验证员 - 复现创意、验证一致性                              │       │
│  │ • 💡 元分析员 - 提出新创作方向                                  │       │
│  └─────────────────────────────────────────────────────────────────┘       │
│                                                                             │
│  2. 负面结果知识库                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐       │
│  │ 记录:                                                           │       │
│  │ • 哪些创意套路已经用烂了                                        │       │
│  │ • 哪些组合已经被证明无效                                         │       │
│  │ • 哪些方向值得探索                                              │       │
│  └─────────────────────────────────────────────────────────────────┘       │
│                                                                             │
│  3. 迭代进化                                                               │
│  ┌─────────────────────────────────────────────────────────────────┐       │
│  │ • 评估 → 优化 → 验证 → 确认                                     │       │
│  │ • 多种子验证 (多角度审查)                                       │       │
│  │ • 独立确认 (不同 Agent 独立验证)                                │       │
│  └─────────────────────────────────────────────────────────────────┘       │
│                                                                             │
│  4. 集体记忆                                                               │
│  ┌─────────────────────────────────────────────────────────────────┐       │
│  │ • 共享实验结果                                                  │       │
│  │ • 从最优配置出发                                                │       │
│  │ • 加速创意发现                                                  │       │
│  └─────────────────────────────────────────────────────────────────┘       │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 新增 Agent 角色

| 角色 | 职责 | 对应 Autoresearch |
|------|------|-------------------|
| 📊 统计员 | 测量创意质量方差、置信度 | Statistician |
| 🔄 验证员 | 复现创意、验证一致性 | Validator |
| 💡 元分析员 | 提出新创作方向 | Meta-analyst |

### 十大发现 → 十大创作原则

```
1. 更多迭代 > 一次完美
2. 简单结构 > 复杂设计
3. 初始创意 > 后期调整
4. 可变 > 固定
5. 小而精 > 大而全
6. 验证 > 假设
7. 敢于否定
8. 关注数据(素材)
9. 共享记忆
10. 不择手段
```
