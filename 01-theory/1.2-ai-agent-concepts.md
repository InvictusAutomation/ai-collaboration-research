# 1.2 AI Agent 的核心概念

## 什么是 AI Agent？

AI Agent（智能体）是一种能够自主理解目标、规划执行路径、执行具体行动的智能系统。与传统的"输入-输出"式 AI 不同，Agent 具有以下核心特征：

## Agent 的能力层级

根据自主能力的大小，AI Agent 可以分为不同的层级：

| 层级 | 名称 | 描述 | 典型场景 |
|-----|------|------|---------|
| L0 | 工具型 | 被动响应指令，无自主决策 | API 调用、模板生成 |
| L1 | 响应型 | 根据明确指令执行任务 | 代码补全、简单问答 |
| L2 | 规划型 | 能够分解任务并制定计划 | 功能开发、问题诊断 |
| L3 | 适应型 | 能够根据反馈调整策略 | 复杂问题解决、迭代优化 |
| L4 | 自主型 | 能够在最小干预下完成复杂任务 | 全栈开发、系统运维 |
| L5 | 协作型 | 能够与其他 Agent 协同工作 | 多 Agent 项目、多人协作 |

### L0-L1: 基础工具层

```python
# L0: 工具型示例 - 简单 API 调用
def get_weather(city: str) -> str:
    """获取城市天气"""
    return f"Weather in {city}: Sunny, 25°C"

# L1: 响应型示例 - 根据指令执行
def process_request(request: str) -> str:
    if "weather" in request:
        city = extract_city(request)
        return get_weather(city)
```

### L2-L3: 规划与适应层

```python
# L2: 规划型示例
class PlannerAgent:
    def plan(self, goal: str) -> List[Task]:
        # 分解任务
        tasks = []
        if "create website" in goal:
            tasks.extend([
                Task("design UI", priority=1),
                Task("setup backend", priority=2),
                Task("implement features", priority=3),
                Task("deploy", priority=4)
            ])
        return tasks

# L3: 适应型示例
class AdaptiveAgent:
    def execute_with_feedback(self, task, feedback):
        if feedback.accuracy < 0.8:
            self.adjust_strategy()
            self.retry()
        return self.final_result
```

### L4-L5: 自主与协作层

```python
# L4: 自主型示例
class AutonomousAgent:
    def accomplish(self, goal: str):
        # 自主理解目标

```

**记忆类型详解**：

| 记忆类型 | 容量 | 持久性 | 访问速度 | 用途 |
|---------|------|--------|---------|------|

## Agent Loop 核心模式

### 基础 Loop 结构

```python
class AgentLoop:
    """基础 Agent Loop"""
    
    def __init__(self, agent: Agent):
        self.agent = agent
        self.max_iterations = 10
    
    async def run(self, task: Task) -> Result:
        for i in range(self.max_iterations):
            # 1. 思考
            thought = await self.agent.think(task)
            
            # 2. 决定行动
            action = await self.agent.decide(thought)
            
            # 3. 执行
            result = await self.agent.execute(action)
            
            # 4. 评估
            if self.agent.evaluate(result):
                return result
            
            # 5. 反思并继续
            await self.agent.reflect(result)
        
        return self.agent.give_up()
```

### Anthropic 提出的六大工作流模式

> 参考来源：[Anthropic - Building Effective AI Agents](https://www.anthropic.com/engineering/building-effective-agents)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Agent 工作流模式分类                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  1. Prompt Chaining (提示链)                                        │
│     输入 → LLM1 → 输出1 → LLM2 → 输出2 → ... → 最终输出             │
│     适用：任务可清晰分解为固定步骤的场景                               │
│                                                                     │
│  2. Routing (路由)                                                  │
│     输入 → 分类器 → 专用处理 → 结果                                  │
│     适用：复杂任务有不同类别，需分别处理                               │
│                                                                     │
│  3. Parallelization (并行化)                                         │
│     输入 → [LLM1, LLM2, LLM3, ...] → 结果聚合                      │
│     适用：子任务相互独立，可并行处理                                  │
│                                                                     │
│  4. Orchestrator-Workers (编排器-工作者)                            │
│     输入 → 编排器 → 动态分配 → 多个工作者 → 结果整合                │
│     适用：无法预先确定子任务的复杂任务                                │
│                                                                     │
│  5. Evaluator-Optimizer (评估器-优化器)                             │
│     输入 → 生成器 → 评估器 ↔ 优化 → ... → 最终结果                  │
│     适用：有明确评估标准，可迭代优化的任务                            │
│                                                                     │
│  6. Autonomous Agent (自主 Agent)                                   │
│     Agent ←→ 工具 ←→ 环境 ←→ 反馈                                   │
│     适用：开放性问题，难以预测所需步骤                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 本章小结

AI Agent 是人机协作的核心载体。理解 Agent 的：
- **能力层级**（L0-L5）帮助我们选择合适的自动化程度
- **核心组件**（记忆、工具、规划器）帮助我们设计系统架构
- **工作流模式**（六大模式）帮助我们选择合适的协作方式
