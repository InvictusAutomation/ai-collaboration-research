## 什么是多智能体系统？

## 多智能体协作的核心概念

### 1. Agent 角色定义

在多智能体系统中，不同的 Agent 通常扮演不同的角色：

| 角色类型 | 职责 | 特点 |
|---------|------|------|
| **Orchestrator** | 任务分发、流程控制 | 全局视角、决策能力强 |
| **Specialist** | 特定领域专家 | 深度知识、执行精确 |
| **Critic** | 质量把控、问题发现 | 独立判断、严格标准 |
| **Coordinator** | 任务协调、资源调度 | 沟通能力、优化能力 |
| **Executor** | 具体执行、结果产出 | 执行力强、效率高 |
| **Monitor** | 状态追踪、异常告警 | 观察细致、反应快速 |

> 参考来源：[Anthropic - Multi-Agent Research System](https://www.anthropic.com/engineering/multi-agent-research-system)

### Role-Playing 模式

CAMEL 的核心创新是"角色扮演"协作模式：

```python
from camel.societies import RolePlaying

# 创建角色扮演社会
society = RolePlaying(
    assistant_role="Python Developer",
    user_role="Product Manager",
    task="创建一个电商网站后端 API"
)

# 运行对话
chat_history = society.initiate_chat()
```

### CAMEL 设计原则

| 原则 | 说明 |
|------|------|
| **Evolvability** | 通过与环境交互持续学习和进化 |
| **Scalability** | 支持百万级 Agent 规模 |
| **Statefulness** | 保持状态记忆，支持多步骤交互 |
| **Code-as-Prompt** | 代码即提示，清晰可读 |

## 通信协议

### 消息格式标准

```json
{
  "message_id": "msg_001",
  "sender": "agent_orchestrator",
  "receiver": "agent_coder",
  "message_type": "task",
  "content": {
    "task_type": "code_generation",
    "task_id": "task_123",
    "requirements": {
      "language": "python",
      "framework": "fastapi"
    }
  },
  "timestamp": "2025-03-13T10:30:00Z"
}
```

### 消息类型

| 类型 | 描述 | 典型用途 |
|-----|------|---------|
| `task` | 任务分配 | 分配具体工作 |
| `result` | 结果返回 | 提交工作成果 |
| `query` | 信息查询 | 请求其他 Agent 的知识 |
| `confirm` | 确认消息 | 确认收到或完成 |
| `alert` | 告警消息 | 发现问题需要关注 |
